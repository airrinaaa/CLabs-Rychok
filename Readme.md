# Менеджер занять (Class Manager)

## Опис проєкту
Цей застосунок на .NET MAUI розроблено для обліку навчальних предметів (сутності першого рівня) та пов'язаних із ними занять (сутності другого рівня). Застосунок реалізує повний цикл роботи з даними: створення, редагування, перегляд, фільтрація, сортування та збереження у базі даних.

## Архітектура та принципи
Проєкт реалізовано як багатошаровий застосунок:
* **UI (MAUI)** — сторінки та ViewModel, побудовані за шаблоном MVVM.
* **Services** — сервіси та DTO, які реалізують бізнес-логіку застосунку.
* **Repositories + Storage** — робота зі сховищем даних SQLite.
* **DBModels** (`SubjectDBModel`, `LessonDBModel`) — моделі для зберігання даних у базі.

UI працює через сервісний шар і не містить прямої логіки доступу до даних. Архітектура побудована з урахуванням принципів **Single Responsibility**, **Inversion of Control** та **Dependency Injection**.

## Логіка роботи застосунку
1. **Головна сторінка**: після запуску відкривається список усіх предметів.
2. **Робота з предметами**: користувач може створювати, редагувати, видаляти предмети, а також шукати, фільтрувати і сортувати їх.
3. **Перегляд деталей предмета**: при відкритті предмета показуються всі його основні поля та список пов'язаних занять.
4. **Робота із заняттями**: користувач може створювати, редагувати, видаляти заняття, переглядати їх деталі, а також сортувати і фільтрувати список.
5. **Збереження даних**: усі зміни автоматично зберігаються у SQLite-базі.

### Реалізовані сторінки
* **SubjectsPage** — головна сторінка, що відображає список усіх предметів.
* **SubjectDetailsPage** — сторінка деталей вибраного предмета, де показано його основні поля та список пов’язаних занять.
* **LessonDetailsPage** — сторінка деталей конкретного заняття з повною інформацією про дату, тип, час початку, час завершення та тривалість.
* **Create/Edit сторінки** — сторінки для створення та редагування предметів і занять.

### Навігація
Навігація між сторінками реалізована через **Shell**:
* зі списку предметів можна перейти до деталей предмета;
* зі сторінки предмета можна перейти до деталей конкретного заняття;
* користувач може повертатися назад до попередніх сторінок стандартними засобами навігації.

Застосунок використовує одне основне вікно.

### Сховище даних
* використовується **SQLite**;
* база даних створюється автоматично при першому запуску;
* при першому запуску додаються початкові дані;
* при видаленні предмета автоматично видаляються всі пов'язані з ним заняття.

### Асинхронність
* вся робота зі сховищем асинхронна;
* під час операцій UI блокується через `IsBusy`;
* відображається індикатор завантаження.

### DI та IoC
У проєкті використано **Dependency Injection** та **Inversion of Control**:
* сервіси, репозиторії та сховище реєструються в DI-контейнері;
* ViewModel отримують залежності через конструктор;
* UI взаємодіє з даними через сервіси.
