# Менеджер занять (Class Manager) — Лабораторна робота №2

## Опис проєкту
Цей застосунок на базі **.NET MAUI** призначений для обліку навчальних предметів та занять. Проєкт демонструє перехід від консольного інтерфейсу до графічного (UI) з дотриманням принципів чистої архітектури та Dependency Injection.

## Основні можливості
* **Кросплатформність**: Підтримка Android та Desktop (macOS/Windows).
* **Трьохрівнева навігація**: Головна сторінка зі списком предметів -> Деталі предмета -> Деталі конкретного заняття.
* **Керування залежностями**: Використання вбудованого IoC-контейнера для реєстрації сервісів та сторінок.

## Архітектура та принципи
* **Inversion of Control (IoC)**: Вся логіка роботи зі сховищем інкапсульована за інтерфейсом `IStorageService`.
* **Dependency Injection (DI)**: Сервіси впроваджуються в сторінки через конструктор або системний сервіс-провайдер.
* **UIModels**: Використовуються для зв'язування даних (Data Binding) з інтерфейсом XAML. Обчислювані поля (наприклад, тривалість занять) реалізовані саме тут.

## Навігаційна модель
Застосунок побудований на базі `AppShell`. Навігація здійснюється за допомогою маршрутів:
* `SubjectsPage` — корінь навігації.
* `SubjectDetailsPage` — перехід при виборі предмета.
* `LessonDetailsPage` — перехід при виборі конкретного заняття зі списку.
Реалізовано можливість зручного повернення назад (Back Navigation) зі збереженням контексту.

## Як запустити
1. Відкрийте рішення `ClassManager.sln` у Visual Studio або VS Code.
2. Виберіть цільову платформу (Android Emulator або MacCatalyst/Windows).
3. Натисніть "Run" (F5).
