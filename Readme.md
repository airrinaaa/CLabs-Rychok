# Менеджер занять (Class Manager)

## Опис проєкту
Цей консольний застосунок розроблено для обліку навчальних предметів (сутності першого рівня) та пов'язаних із ними занять (сутності другого рівня). Проєкт закладає базову об'єктну модель для майбутнього застосунку з дотриманням принципів багатошарової архітектури.

## Архітектура та принципи (Single Responsibility)
Для кожної сутності створено по два класи з чітко розділеними обов'язками:
* **DBModels** (`SubjectDBModel`, `LessonDBModel`): Бібліотека класів, які відповідають винятково за зберігання даних у базі. Згідно з вимогами, вони не містять обчислюваних полів. Також об'єкти першого рівня (`SubjectDBModel`) не містять у собі колекцій об'єктів другого рівня, щоб уникнути жорсткої зв'язності.
* **UIModels** (`SubjectUIModel`, `LessonUIModel`): Бібліотека класів, які відповідають за створення, відображення та редагування сутностей. Саме в цих класах реалізована бізнес-логіка: обчислювані поля (тривалість окремого заняття `LessonDuration` та загальна тривалість предмета `DurationAll`), а також колекція занять для конкретного предмета.
* **Services**: 
  * `FakeStorage`: Статичний клас, що слугує штучним сховищем. При першому зверненні він заповнюється реалістичними початковими даними: 3 предмети та 14 занять.
  * `StorageService`: Клас, який інкапсулює логіку взаємодії зі штучним сховищем та використовується для отримання об'єктів зберігання.

## Логіка роботи застосунку (ConsoleApp)
1. **Головне меню**: Після запуску програма через `StorageService` отримує всі предмети, перетворює їх на `UIModels` та виводить список на екран.
2. **Ліниве завантаження (Lazy Loading)**: З метою оптимізації, заняття (сутності другого рівня) не завантажуються з бази даних відразу. Тому початково біля кожного предмета відображається `Lessons: 0`.
3. **Перегляд деталей**: Коли користувач вводить назву конкретного предмета (без чутливості до регістру), застосунок звертається до сховища, завантажує пов'язані заняття для цього екземпляра та обчислює загальну тривалість.
4. **Навігація**: Користувач може повернутися до головного списку командою `Back` (при цьому завантажені заняття зберігаються в пам'яті моделі) або штатно завершити роботу командою `Exit`.
